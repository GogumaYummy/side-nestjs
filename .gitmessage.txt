# 머릿말, 본문, 꼬릿말로 구성
# 머릿말 외에는 생략 가능
# 각 구역은 비어 있는 줄로 구분
#
# 머릿말
#   <타입>: <제목> 형식
#   제목은 50자 이내로 무엇을 했는지 간단하고 명확하게 작성
#   타입:
#     feat - 새로운 코드 작성
#     fix - 문제 코드 또는 오타 수정
#     docs - 문서 작성 또는 수정
#     test - 테스트 코드 작성
#     refactor - 코드 리팩터링
#     style - 코드 스타일 적용
#     chore - 패키지 매니저, 빌드 정보 수정
#
# 본문
#   커밋 내용에 대한 설명을 최대한 자세히 작성
#   어떻게보다는 무엇을 왜 변경했는지 작성
#   한 문단이 72자를 넘지 않도록 작성
#
# 꼬릿말
#   이슈에 관한 커밋일 경우 이슈 트래커 ID를 작성
#   <키워드>: #<이슈 번호> 형식으로 작성
####################
